4. ARCHITECTURE
================================================================================

  Single-object architecture: obj_game
  Three events handle everything:

  CREATE_0 (initialization)
    - Calls init_all_data() to load all JSON data into globals
    - Initializes portrait system
    - Creates blank hero statblock
    - Sets up UI state (tabs, scroll, animation, GM mode)
    - Configures window mode and GUI size from config
    - Builds chargen dropdown system
    - Loads saved campaign
    - Applies colorblind mode

  STEP_0 (input processing)
    - Reads mouse click + scroll wheel
    - Global exit handler (saves hero + ends game)
    - Roll animation timer
    - State machine: welcome → chargen → skill browser → main screen
    - Main screen routes to: GM tools (gm_state="gm") or player sheet
    - Right panel buttons always active (new char, menu, portrait, file ops,
      situation die, roll, quick rolls)
    - Tab handlers dispatched by current_tab (0-9)
    - GM tab handlers dispatched by gm_tab (0-4)

  DRAW_64 (rendering)
    - Gradient background with wave swoop
    - Welcome screen OR character sheet + right panel
    - Left panel: tabs (vertical or horizontal) + content area
    - Tab content dispatched to draw_tab_* functions
    - Right panel: portrait, dice roller, roll history, file ops
    - Overlays: skill browser, chargen popup, accessibility settings, tooltips
    - Button rects registered via btn struct each frame

  Script hierarchy:
    scr_ui_helpers     Core UI (btn_clicked, meta_roll, dropdowns, draw helpers)
    scr_data_loader    JSON pipeline (init, read/write, campaign persistence)
    scr_alternity_statblock  Character data (create, update, export/import, portraits)
    scr_draw_tabs      Tab renderers (10 player + 5 GM + helper functions)
    scr_tab_handlers   Tab input (10 player + 4 GM handlers)
    scr_skill_costs    Skill economy (costs, FX management, rank manipulation)
    scr_chargen        Character generation (species, careers, NPC templates)
    scr_dice           Dice engine (core checks, situation die)

5. CODE STYLE CONVENTIONS
================================================================================

  - All local variables prefixed with underscore: _variableName
  - Verbose descriptive camelCase for variable names
  - Single-letter vars only for loop counters: _i, _j, _k
  - Static-length content first, variable-length content last in scrollable views
  - REMOVED comments mark deleted dead code (e.g., "// shuffle_array() REMOVED")
  - Functions named with snake_case (GML convention)
  - Structs used extensively for data (statblocks, config, FX entries)
  - Button registration pattern: Draw registers rect, Step checks click
  - Tab functions return updated _ly position for vertical layout

6. KEY PATTERNS
================================================================================

  btn_clicked(key)
  ----------------
  Draw_64 registers button rects each frame:
    btn.myButton = [x1, y1, x2, y2]   (via variable_struct_set or ui_btn)
  Step_0 checks if mouse clicked inside:
    if (btn_clicked("myButton")) { ... }
  ui_btn() combines registration + drawing in one call.

  meta_roll(stat, request)
  ------------------------
  THE single entry point for all dice checks. Flow:
    1. Caller builds roll_request via build_*_request()
    2. prepare_roll() applies auto-mods (training, FX, wounds)
    3. Sets staged_roll with computed situation step
    4. Player can adjust die with < > buttons
    5. execute_staged_roll() rolls dice, applies cant-fail, logs result
  For instant rolls: meta_roll() = prepare_roll() + execute_staged_roll()
  For quick rolls: quick_roll_skill() adds auto-mods to player's current die.

  update_hero(stat)
  -----------------
  MUST be called after ANY character modification. Recalculates:
    - Ability derived values (untrained, resistance modifier)
    - Action check (DEX+INT/2 + profession bonus)
    - Durability (preserves current damage)
    - Skill scores (all skills recalculated from base abilities)
    - Portrait (loads appropriate image)

  skill_index_map
  ---------------
  Maps visible filtered rows to hero.skills[] indices. Rebuilt each frame
  during draw_tab_character(). Used by handle_tab_character() for click/+/-
  operations on the correct skill when filtered by active_stat_group.

  apply_fx_modifiers(stat, request)
  ---------------------------------
  THE unified modifier engine. One loop through all active FX on hero:
    1. Generates keyword tags for the roll (ability, type, training, skill names)
    2. Looks up each FX in global.fx_database
    3. Matches FX keywords against roll tags
    4. Applies modifier based on quality tier or fixed value
  Three modes: keyword_tiers (per-quality keywords), quality_scale (same keywords,
  different modifier per quality), fixed (flat keywords + flat modifier).
  Adding new modifiers requires ZERO code changes — just add to fx_database.json.

  Dropdown System
  ---------------
  Self-contained dropdown structs with draw(), draw_list(), click(), value(),
  set_options() methods. Used by chargen popup. dropdown_group_click() and
  dropdown_group_draw_lists() manage groups of dropdowns.

  keyword_tree.json  Maps skills → abilities → categories for modifier matching
  equipment.json     Weapons (ranged/melee/heavy), armor, gear — all PL-filtered
  programs.json      Grid programs, computers, processor qualities
  npc_templates.json GMG NPC templates (abilities, skills, equipment)
  changelog.json     Development log entries

  Character Save Format (v4)
  ---------------------------
  Compact JSON:
    _format: "alternity_statblock", _version: 4
    Skills as arrays: [name, ability/broad, rank/-1]
    Weapons as 7-element arrays: [name, skill_kw, dmg_o, dmg_g, dmg_a, range, type]
    Abilities as uppercase keys: STR, DEX, CON, INT, WIL, PER
    Durability as damage taken: stun_damage, wound_damage, mortal_damage
    FX array: [{name, type, quality, active}]
    Deck: {computer, processor, programs[{name, quality}]}
    Lore: {height, weight, hair, gender, moral_attitude, temperament[],
           motivations[], personality, lifepath}
  Import handles v2 (nested objects), v3 (lowercase), v4 (compact arrays).

  Campaign Persistence
  --------------------
  campaign.json stores:
    - factions: array of faction/team names
    - roster: array of {name, path} for all known character files
    - party: array of {name, path} for active party members
    - npcs: array of {name, path, faction} for NPCs
  Characters save individually to working_directory + "characters/*.json"
  recent_characters.json tracks last character and recent list (cap 10).

8. GM MODE
================================================================================

  Toggle: gm_mode (bool). Activated from welcome screen.
  States: gm_state = "gm" (GM tools) or "edit" (editing a character)
  When gm_state="gm", replaces the entire left panel with GM tabs.
  "Edit" enters the normal character sheet for the selected character.
  "Back to GM" button returns to GM tools.

  GM Tabs (gm_tab 0-4):
    0 = Party     Party roster with HP, senses, resolve, resistance mods,
                  enhanced senses, psionics. Edit/Move/Remove buttons.
    1 = NPCs      NPCs organized by faction/team. Quick NPC generation from
                  GMG templates. Import, add faction, edit/move/remove.
    2 = Encounter Initiative bar (4 phases: Amazing/Good/Ordinary/Marginal).
                  Per-character AC, phase, actions, resistance mods, durability.
                  Roll All Init, New Round, Clear Init buttons.
    3 = Campaign  Master roster of all character files. Scan directory, import,
                  export all. Load characters to party or NPCs.
    4 = Resources Scrollable GM customization guide — how to edit equipment,
                  skills, FX, templates, species JSON files.

  Tab 2: Combat
    Quick roll buttons (Senses, Mental Res, Physical Res, Initiative).
    Action phase tracker (4 phases with placement). Wound/cant-fail/range toggles.
    Weapon attack table with skill matching, training, situation die, damage codes.
    Post-success damage prompt. Unarmed combat (punch/kick). Defensive skills
    (Dodge, Power martial arts). Feat checks (all 6 abilities).

  Tab 3: Psionics
    Psionic disciplines (Telepathy, Telekinesis, ESP, Biokinesis). Broad and
    specialty skills with descriptions. Roll buttons. Wound/cant-fail toggles.

  Tab 4: Perks & Flaws
    Perks/Flaws sub-tabs. Your installed perks/flaws with cost display.
    Available list from fx_database with scroll. Quality tier upgrades (O/G/A).
    Click to add/toggle.

  Tab 5: Cybertech
    Cyber tolerance display (CON + species + perk bonuses). Installed cyberware
    with active/inactive toggles. Available cyberware by category
    (neural/body/sensory/weapon/utility). Expandable details with quality tiers.
    Auto-prerequisite installation.

  Tab 6: Roll Log
    Persistent roll log from file. Last 100 entries, newest first. Color-coded
    by degree (Amazing/Good/Ordinary/Failure). Copy-to-clipboard buttons.
    Scroll up/down.

  Tab 7: Info
    Scrollable help content. In player mode: how-to-use guide for all tabs.
    In GM mode: GM resources and JSON customization guide.

  Tab 8: Grid / Cyberdeck
    Programs sub-tab: installed programs with slots display. Add from database
    filtered by type (operator/hacking/utility) and PL.
    Computer sub-tab: select cyberdeck hardware with processor quality.
    Builder sub-tab: create custom homebrew programs.

  Tab 9: Aura / Lore
    Character identity (species, profession, gender, height, weight, hair).
    PHB Step 8 attributes: moral attitude, temperament (pill buttons, pick 2-3),
    motivations (pill buttons, pick 1-2), personality (freeform), lifepath
    (freeform backstory). Sergeant Voss is read-only template.
