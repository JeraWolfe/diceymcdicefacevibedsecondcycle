=== v0.15.1: PHB Skill Cost Verification ===
Went through the PHB with a fine-tooth comb. Page 36 has the magic formula: 'The cost to purchase a higher rank in a specialty skill is either the list price or the list price -1, plus a number of points equal to the character's current rank.' So rank 3 in a profession specialty costs 1+2+3=6 points total, not 3. That escalation makes high-rank specialties genuinely expensive. The old flat-rate system was way too generous. Also discovered the PHB only has two cost tiers (your profession or not), not three. 'Related' was my invention. Keeping it for display but costs match 'Other' now. - Dev

=== v0.32.0: Unified FX Engine & Keyword Tree — The Great Refactor Begins ===
This is the big one. The refactor that changes how the engine thinks. Before today, perks were one system, flaws were another, cybertech was a third, and they all did the same thing — modify your rolls based on keywords — but through three completely different code paths. Observant checked a hardcoded keyword map. Fast Chip checked an if-statement. Clumsy checked a different hardcoded map. A GM wanting to homebrew a new perk had to edit GML. That ends now. One FX struct. One database. One matching function. Perks, flaws, cybertech, racial traits, environmental hazards, magic items — they're all the same thing with a type field. The keyword tree is the foundation: every skill in the game mapped to its ability and categories, auto-generated from the skill tree so it never gets out of sync. And the feat check fix — that's been wrong since version 0.2. The PHB is clear: feat checks use FULL ability score. We were halving it. Voss's STR 12 was rolling against 6 on feat checks when it should have been 12. That's a massive difference. Fixed now, fixed forever. The nuclear fuel from the extraction hasn't worn off. We're rebuilding the engine while it's running. Seven phases. One FX struct to rule them all. - Dev
